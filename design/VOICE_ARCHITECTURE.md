# Voice Architecture

## The Core Idea

The student never goes to a settings page to change the voice. They just say it — or type it — in the chat:

> "Can you sound more like a calm female professor?"
> "Make your voice deeper"
> "Speak with a British accent"
> "You sound too robotic, change it"
> "Be more energetic"

The tutor changes its voice immediately and speaks the confirmation in the new voice so the student hears the change right away.

---

## How It Works (Full Flow)

```
Student says: "sound more like a female professor with a British accent"
        │
        ▼
Agent Manager detects voice change intent
(detectVoiceChangeIntent() in voice-service.ts)
        │
        ▼
Claude translates request → ElevenLabs Voice Design prompt
"Female, 35-45. British RP accent. Persona: university professor.
 Emotion: warm, authoritative, encouraging. Clear articulation,
 measured pace, forward presence."
        │
        ▼
ElevenLabs Voice Design API generates the voice
(new voice_id created in the student's ElevenLabs account)
        │
        ▼
voice_id saved to neuro.voice in Brain DB
(persists across sessions — the tutor always uses this voice)
        │
        ▼
Tutor speaks confirmation in the NEW voice:
"There we go — how does this sound? I can keep adjusting if you want."
        │
        ▼
All future TTS uses the new voice_id automatically
```

## Streaming Pipeline (How We Get <1 Second Latency)

### Old approach (current — 2-3 second delay):
```
Full LLM response ready → Send entire text to ElevenLabs → Wait for full audio → Play
```

### New approach (streaming — <1 second to first audio):
```
LLM streams tokens → First sentence complete → Send to ElevenLabs immediately
                                                        │
                                                        ▼
                                              Audio for sentence 1 ready (~300ms)
                                                        │
                                                        ▼
                                              Play sentence 1 ← student hears this
                                                        │
                                              (while playing sentence 1)
                                                        ▼
                                              Send sentence 2 to ElevenLabs
                                                        │
                                                        ▼
                                              Audio for sentence 2 ready
                                                        │
                                                        ▼
                                              Play sentence 2 seamlessly
```

**Result:** Student hears the first sentence in ~800ms total (300ms LLM first token + 300ms ElevenLabs turbo + 200ms network). The rest plays seamlessly while the first sentence is already playing.

## Voice Model

| Model | Latency | Quality | Use Case |
|---|---|---|---|
| `eleven_turbo_v2_5` | ~300ms | High | **Use this — real-time conversation** |
| `eleven_multilingual_v2` | ~800ms | Highest | Long-form content, not real-time |
| `eleven_monolingual_v1` | ~600ms | Good | English only, legacy |

**Always use `eleven_turbo_v2_5` for the tutor chat.**

## Voice Storage in Brain DB

The student's voice preference is stored in `neuro.voice`:

```sql
neuro.voice
  person_id         UUID  — links to neuro.persons
  voice_id          TEXT  — ElevenLabs voice_id
  voice_description TEXT  — the Voice Design prompt used
  student_request   TEXT  — what the student originally asked for
  model             TEXT  — always 'eleven_turbo_v2_5'
  updated_at        TIMESTAMPTZ
```

This is part of the brain's memory. The tutor always sounds the way the student wants it to sound, across every session, on every device.

## What the Frontend Needs to Build

### 1. Replace `/api/tts` with `/api/voice/tts-stream`

The current frontend calls `/api/tts` and waits for the full audio. Replace with:

```javascript
// OLD (2-3 second delay)
const { audio } = await fetch('/api/tts', { body: { text } }).then(r => r.json());
playAudio(audio);

// NEW (first sentence plays in <1 second)
const eventSource = new EventSource('/api/voice/tts-stream');
// Actually use fetch with streaming:
const response = await fetch('/api/voice/tts-stream', {
  method: 'POST',
  body: JSON.stringify({ text, person_id }),
});

const reader = response.body.getReader();
const audioQueue = [];
let isPlaying = false;

while (true) {
  const { done, value } = await reader.read();
  if (done) break;

  const event = JSON.parse(new TextDecoder().decode(value));
  if (event.type === 'audio_chunk') {
    audioQueue.push(event.audio);
    if (!isPlaying) {
      isPlaying = true;
      playAudioQueue(audioQueue); // starts playing immediately
    }
  }
}
```

### 2. Add Voice Change Detection in Chat

When the student sends a message, check for voice change intent before sending to the main chat API:

```javascript
import { detectVoiceChangeIntent } from '../utils/voice-utils';

const handleSendMessage = async (message) => {
  if (detectVoiceChangeIntent(message)) {
    // Route to voice change flow
    const result = await fetch('/api/voice/change', {
      method: 'POST',
      body: JSON.stringify({ person_id, request: message, tutor_name }),
    }).then(r => r.json());

    // Update local voice_id state
    setCurrentVoiceId(result.voice_id);

    // Play the confirmation in the new voice
    playTTS(result.confirmation_text, result.voice_id);

    // Show preset options in chat
    showVoicePresets();
    return;
  }

  // Normal chat flow
  sendToAgentManager(message);
};
```

### 3. Voice Preset Panel

When voice is mentioned, show a horizontal scroll of 8 preset buttons in the chat:

```
[😊 Friendly] [🎓 Professor] [⚡ Energetic] [🧘 Calm] [🇬🇧 British] [🎙️ Deep] ...
```

Tapping a preset immediately switches the voice and plays a short confirmation.

### 4. "Speaking…" Indicator

The Neural Ring already has a `.nr-speaking` CSS class that pulses when the tutor is speaking. Wire this to the streaming state:

```javascript
// Start pulsing when first audio chunk arrives
onFirstChunk: () => neuralRing.classList.add('nr-speaking'),
// Stop when all chunks are done
onDone: () => neuralRing.classList.remove('nr-speaking'),
```

## Voice Change as a Brain Signal

Every voice change is recorded as a `preference` signal in `brain.signals`. Over time, the brain learns:
- What kind of voice the student prefers
- How often they change it (students who change voice often may be experimenting with their learning style)
- Whether certain voice types correlate with longer study sessions

This data feeds into the Reflection Engine and eventually the Situation Synthesizer, which can proactively suggest a voice change: *"You tend to study longer with a calmer voice — want me to switch to your calm mode?"*
