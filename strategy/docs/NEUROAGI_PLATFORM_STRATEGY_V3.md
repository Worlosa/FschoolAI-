## The Thesis

Every human being's brain is completely disconnected from their digital and physical world. That disconnection is the bottleneck on human potential.

NeuroAGI is building the personal layers of intelligence that general AI will never build. We sit above the Universal Model (GPT-4, Claude) and below the human — fusing physical context (Neural Card), digital behaviour (FschoolAI), and cognitive patterns (Brain OS) into a sovereign, compounding model of one specific person's world and mind.

> **"Your brain. Your world."**

This is not an EdTech product. It is not a second brain app. It is not a smarter assistant. It is a **Personal World Model** — a persistent, sovereign AI that models one specific human's reality by fusing their physical and digital context. Students are v1 because they have the most structured context and the most measurable outcomes. But the architecture is human-specific, not student-specific.

## The Ecosystem Model — iOS for Human Intelligence

NeuroAGI is a platform company with an ecosystem, not just a product company.

```
NeuroAGI Brain OS (Personal World Model — sovereign, owned by user)
        ↓
FschoolAI (First-party app — proves the model works for students)
        ↓
Third-party AGI apps (startups, developers, platforms)
        ↓
AI-enabled hardware (Neural Card → Chinese AI hardware ecosystem)
        ↓
All running on the same brain — owned by the user
```

This is the Apple iOS model applied to human intelligence. Apple built the iPhone (hardware + OS), then built the first apps to prove the platform works, then opened the App Store. Every app on iOS runs on the same device, same OS, same identity layer.

NeuroAGI builds the Brain OS. FschoolAI is the first app. The developer platform opens in Phase 3. Chinese AI hardware partners integrate in Phase 4. The brain is the centre of all of it. The user owns the brain.

### Why Developers Will Build on NeuroAGI

Right now, building an AI agent has no technical entry barrier — anyone can wrap Claude in a UI. But three problems kill most agents:
1. **No memory** — the agent resets every session, knows nothing about the user
2. **No trust** — users do not know if this random startup's agent is safe
3. **No distribution** — the agent is unknown, marketing is expensive

NeuroAGI solves all three at the platform level. Developers get rich, real-world context that no other platform provides. Users get agents that actually know them. NeuroAGI takes a platform fee.

## The Hardware Layer — Neural Card + Chinese AI Ecosystem

### Neural Card
The Neural Card is the physical sensor layer of the Personal World Model. It attaches to an iPhone via MagSafe. On-device processing via Whisper — raw audio never leaves the device. Only abstracted signals are written to the brain.

**Why hardware matters:** The digital layer tells you what the student is doing. The physical layer tells you how they are doing it and in what state. The fusion of both is the Personal World Model.

### Chinese AI Hardware Ecosystem
China is building AI into every physical product — Haier, Midea, Xiaomi, TCL. The problem: every device has its own app, its own cloud, its own data silo. NeuroAGI's position: the Brain OS becomes the unified intelligence layer that every AI-enabled device connects to. Aurora Chen's supply chain network and Alibaba leadership relationships are the direct path to these partnerships.

## The Privacy Architecture

The privacy model is not a feature — it is the reason the ecosystem is possible at all.

- **On-device processing first:** Raw audio never leaves the Neural Card
- **Transparent permission system:** Users see exactly what each agent accesses
- **DID ownership:** Brain anchored to user's cryptographic key, not a NeuroAGI account

OpenAI and Google cannot offer this because their business model requires owning the data. NeuroAGI's business model is platform fees and hardware — not user data.

## Critical Architecture Principle: Agent-Agnostic Brain

**The brain is not FschoolAI-specific. It is agent-agnostic.**

Every brain table is designed so that any agent — a fitness agent, a professional agent, a creative agent — can read from and write to the same brain. FschoolAI is the first agent that happens to be very good at the academic context.

The brain should not have FschoolAI-specific tables. It should have:
- `signals` (any agent can emit signals)
- `knowledge_graph` (any agent can read and write concepts)
- `credentials` (any agent can issue or verify)
- `permissions` (the user controls which agent sees what)
